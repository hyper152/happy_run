## 一、基本原理

```
┌─────────────────────────────────────┐
│          真机（手机本体）               │
│  ┌──────────────┐                    │
│  │ Fake Location │  ← 提供虚假GPS位置  │
│  └──────┬───────┘                    │
│         │ 位置信息                    │
│         ▼                            │
│  ┌──────────────┐                    │
│  │  光速虚拟机    │  ← 步道乐跑运行在此  │
│  │  (Android 7.1)│                    │
│  │  ┌──────────┐│                    │
│  │  │ 步道乐跑   ││ ← 读取到虚拟位置    │
│  │  └──────────┘│                    │
│  └──────────────┘                    │
└─────────────────────────────────────┘
```

**核心思想**：乐跑APP运行在**虚拟机内**，Fake Location运行在**真机系统**上。乐跑无法穿透虚拟机检测到真机上的模拟位置软件，上报数据时会显示"环境正常"，有效避免"跑步行为不规范"的警告。

## 二、准备工作清单

### ✅ 你需要准备的东西

| 项目 | 说明 |
|------|------|
| 📱 **安卓手机** | 一台能正常开机的安卓手机 |
| 🌐 **网络** | 手机需要联网（WiFi 或 流量均可） |
| 📦 **以下两个 APP** | 见下文安装步骤 |

### 📥 需要安装的 APP

1. **Fake Location**（本仓库已提供：`fl.apk`）
2. **光速虚拟机**（在手机应用商店搜索下载即可，或官网下载）

> ⚠️ 建议选择 **Android 7.1 系统** 的虚拟机版本，稳定且完全免费。

## 三、第一步：安装 Fake Location

### 3.1 安装方法

1. 将 `fl.apk` 文件传输到手机上（可通过微信、QQ、数据线等方式）
2. 在手机上点击安装

### 3.2 ⚠️ 避坑：安装报风险提示怎么办？

安装时手机可能会提示"风险应用"或"安全提醒"，这是正常现象，因为 Fake Location 需要获取修改位置信息的权限。

**解决方法**：
- **断网安装**：先把手机切换到飞行模式（关掉 WiFi 和移动数据），再点击安装
- 安装成功后可以恢复网络连接

### 3.3 授予必要权限

安装完成后，进入手机的 **设置 → 应用管理 → Fake Location**，手动开启以下权限：

| 权限 | 作用 |
|------|------|
| ✅ **后台弹出界面** | 确保模拟位置时能正常弹出控制界面 |
| ✅ **显示悬浮窗** | 可以在其他 APP 上层显示悬浮控制按钮 |

> 📱 **不同手机路径可能不同**：华为/荣耀在"设置→应用→应用管理"；小米在"设置→应用设置→应用管理"；OPPO/vivo 在"设置→应用管理"。

## 四、第二步：安装光速虚拟机

### 4.1 下载与安装

- 在手机自带的应用商店搜索 **"光速虚拟机"** 并安装
- 或者前往官网下载 APK 安装

### 4.2 选择系统版本

首次打开光速虚拟机时，会让你选择系统镜像，请选择：

> ✅ **Android 7.1 系统**（稳定且完全免费，兼容性好）

### 4.3 ⚠️ 重要设置：停止限制子进程

进入虚拟机后，需要打开虚拟机的**开发者选项**，找到并打开 **"停止限制子进程"**。

> **为什么需要这个？** 不开启的话，虚拟机在后台运行一段时间后，手机系统可能会把它杀掉（为了省电），导致跑步中断。

**开启方法**：
1. 在虚拟机桌面找到 **"设置"**
2. 进入 **"开发者选项"**（如果没有看到开发者选项，参考下文"如何打开开发者选项"）
3. 找到 **"停止限制子进程"**，将其开关**打开**

### 4.4 往虚拟机里导入 APP

步道乐跑需要在虚拟机内运行，所以需要把以下 APP 导入虚拟机：

1. **步道乐跑**（从应用商店下载安装包，或从真机导入）
2. **微信 或 QQ**（用于登录乐跑账号）

**导入方法**：在光速虚拟机主界面一般有"导入应用"或"添加应用"按钮，点击后从真机的已安装应用中选择即可。

## 五、第三步：真机环境配置（开发者选项）

> 这一步是在**真机（手机本体）**上操作，不是在虚拟机里。

### 5.1 开启开发者选项

1. 打开手机 **设置**
2. 进入 **关于手机**
3. 连续点击 **"版本号"** 7 次，直到提示"已进入开发者模式"

> 📱 不同机型操作略有不同，如果找不到"版本号"，可以搜索：
> - "手机型号 + 如何打开开发者选项"（例如"小米14如何打开开发者选项"）

### 5.2 ⭐ 关键步骤：设置模拟位置应用

> 🚨 **这一步最重要，很多人在这里出错！**

**操作路径**：
1. 返回 **设置 → 更多设置**（或"系统和更新"）
2. 进入 **开发者选项**
3. **找到并点击** → `选择模拟位置信息应用`
4. **在列表中选中** → **`Fake Location`**

**检查是否成功**：
- 设置完成后，`选择模拟位置信息应用` 这一项应该显示为 **"Fake Location"**
- 如果显示"无"或其他的，说明没设置成功，需要重新操作

> ✅ 设置成功后，Fake Location 就获得了向系统提供虚拟位置的权限，这是整个流程**最关键的环节**。

## 六、第四步：虚拟位移设置（Fake Location）

### 6.1 打开 Fake Location

在真机上打开 Fake Location APP，你会看到地图界面。如果地图不显示，请检查网络连接。

### 6.2 路线模拟（主要功能：跑里程+打打卡点）

路线模拟的作用是：**让位置沿着操场跑道持续移动**，这样乐跑的里程数会上涨，并且路过打卡点时会自动打卡。

**操作步骤**：

1. **打开菜单**：点击左上角的 **☰（三条杠）菜单按钮**
2. **选择"路线模拟"**：在菜单中找到并点击
3. **添加路线点**：点击底部的 **"+"号**
4. **在地图上选点**：
   - 地图上会显示你当前的位置
   - 导航到学校的操场（如重庆大学虎溪校区操场）
   - 在跑道上按顺序点几个点，连成一个**闭合的圈**（建议 4-6 个点即可）
5. **设置速度**：点击下方的 **"小人骑车"图标**，设置移动速度：
   - 建议范围：**3.5 m/s ~ 4.5 m/s**（约等于正常跑步速度）
   - 速度太慢 -> 跑完规定里程需要很久
   - 速度太快 -> 可能被乐跑判定异常
6. **保存路线**：确认路线没有问题后保存

### 6.3 位置模拟（补充功能：补打漏掉的打卡点）

如果你发现路线跑完后还有打卡点没打到，可以使用"位置模拟"来补救。

**操作步骤**：
1. 点击左上角 **☰ 菜单 → "位置模拟"**
2. 使用 **摇杆** 慢慢移动位置，或者直接**点击地图上打卡点的位置**
3. 位置会瞬间移动过去，"吃掉"那个打卡点

### 6.4 启动模拟

1. 点击 **"启动模拟"** 按钮
2. 确认状态：查看手机顶部的通知栏，应该显示 **"正在模拟位置"**
3. ✅ 此时真机已经在向系统提供虚拟位置了

## 七、第五步：虚拟机内跑乐跑

### 7.1 进入虚拟机

1. 打开光速虚拟机
2. 进入 Android 7.1 虚拟机系统

### 7.2 打开步道乐跑

1. 在虚拟机桌面找到 **步道乐跑** APP
2. 打开并登录（使用微信或 QQ 授权登录）
3. 进入 **跑步** 页面

### 7.3 开始跑步

1. 点击乐跑的 **"开始跑步"** 按钮
2. **📌 注意延迟**：位置同步会有 **20 秒左右的延迟**
   - 刚开始跑时，地图上的蓝色位置点可能不会立即移动
   - 这是正常现象，请耐心等待 20-30 秒

### 7.4 ⚠️ 如果蓝点不动怎么办？

如果等待超过 30 秒蓝点还是不动，请尝试以下方法：

**方法一：悬浮窗模式**
- 将 Fake Location 以**小窗模式**或**悬浮窗**方式，悬浮在乐跑跑步页面的上方
- 这样可以不断刷新位置同步

**方法二：切换摇杆模式**
- 回到 Fake Location，切换到"位置模拟"的摇杆模式
- 轻微拨动摇杆，激活位置更新
- 再切换回路线模拟

**方法三：暂停重启**
- 在乐跑中点击 **暂停**，等待 3-5 秒
- 再点击 **继续**
- 通常位置就会开始更新

## 八、第六步：步频处理与结束上传

### 8.1 🏃 步频问题

由于是模拟定位，手机并没有真实的物理位移，所以**计步器可能不会增加步数**。

**目前情况**：步道乐跑对步频的检查比较宽松，通常里程和打卡点达标就能正常上传。

**如果你想更稳一些**，可以：
- 跑步过程中**手动摇晃手机**，模拟真实跑步的晃动
- 或者为虚拟机安装"传感器模拟器"插件（光速虚拟机插件市场可能有）

### 8.2 结束跑步并上传

当**里程数**和**打卡点**都满足乐跑要求后：

1. **先**在虚拟机内点击乐跑的 **"结束跑步"** 按钮
2. 等待上传：点击"结束"后会显示"上传中..."
3. 必须看到 **"上传成功"** 的提示才算完成！
4. **再**关闭虚拟机和 Fake Location

> ⚠️ 如果上传失败，可以尝试重新连接网络再试。不要慌，只要还没关闭模拟位置，重新点击结束跑步一般能成功上传。

## 九、常见问题与避坑指南

### ❓ Q1：乐跑提示"跑步行为不规范"怎么办？

**原因**：乐跑检测到了模拟定位环境。

**解决方案**：
- 确认 Fake Location 是运行在**真机**上，乐跑在**虚拟机**内
- 检查是否给虚拟机开启了"停止限制子进程"
- 确保虚拟机内没有安装任何位置相关的工具

### ❓ Q2：里程数不增加怎么办？

**可能原因**：
1. 位置同步延迟（等待 20-30 秒）
2. Fake Location 未成功启动模拟（检查通知栏是否有"正在模拟位置"）
3. 路线设置不正确（路线没有形成闭合圈？）

### ❓ Q3：打卡点打不上怎么办？

**解决方案**：
1. 使用 Fake Location 的"路线模拟"，确保路线经过打卡点
2. 如果还是漏了，切换到"位置模拟"直接移动到打卡点位置
3. 有些打卡点需要在特定范围内停留几秒，可以适当减慢速度

### ❓ Q4：跑着跑着位置不动了？

**可能原因**：
1. 虚拟机被系统杀掉了 -> 检查"停止限制子进程"是否开启
2. Fake Location 后台被限制 -> 给 Fake Location 开启"后台弹出界面"权限
3. 网络波动 -> 检查网络连接

**解决方案**：暂停乐跑 → 检查 Fake Location 状态 → 重启模拟 → 继续跑步

### ❓ Q5：为什么必须用虚拟机？

直接把 Fake Location 和乐跑都装在本机上不行吗？

**不行**。因为乐跑会扫描手机系统，检测是否安装了模拟定位软件。如果把两个 APP 都装在真机上，乐跑可以直接检测到 Fake Location 的存在，就会判定"环境异常"，跑步记录可能被判定无效甚至封号。

**使用虚拟机后**，乐跑在虚拟机这个"沙盒"里运行，它只能看到虚拟机内部的信息，看不到真机系统上有什么 APP，所以检测不到 Fake Location，上报数据时显示"环境正常"。

### ❓ Q6：上传成绩一直失败怎么办？

1. 检查虚拟机内的网络连接是否正常
2. 尝试关闭再打开乐跑 APP
3. 确认里程和打卡点是否满足要求
4. 如果还是不行，放弃这次记录，重新跑一次

## 十、附录：术语解释

| 术语 | 说明 |
|------|------|
| **真机 / 宿主机** | 你手中的实体手机，运行 Android 系统 |
| **虚拟机** | 在手机上通过 APP 模拟出的"虚拟手机"，在里面可以安装和使用软件 |
| **Fake Location** | 一款模拟定位软件，可以让手机系统认为你在另一个位置 |
| **光速虚拟机** | 一款能在手机上运行虚拟机的 APP，类似电脑上的 VMware |
| **模拟位置** | 让手机系统以为你在 A 点，实际上你在 B 点 |
| **开发者选项** | Android 系统的隐藏功能菜单，供开发者调试使用 |
| **停止限制子进程** | 让虚拟机在后台运行时不会被手机系统杀掉 |
| **打卡点** | 乐跑路线上标记的必须经过的位置点 |
| **路线模拟** | 设定一条路线，让位置沿路线自动移动 |
| **位置模拟** | 直接将位置设定到地图上的某个固定点 |
